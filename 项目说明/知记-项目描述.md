# 知记 (Zhiji) — 项目描述

> 独立开发 · 28 小时从零到上线 · Flutter + DeepSeek API + SQLite/FTS5 + RAG

---

知记是一款面向个人用户的 AI 原生知识管理 Android 应用。用户可以写日记、管理知识、添加附件、全文搜索，通过接入 DeepSeek 大模型实现 AI 自动摘要、续写、润色、总结和 RAG 智能问答。应用采用离线优先架构，所有用户数据存储在设备本地 SQLite 数据库中，不上传云端。

项目使用 Flutter 3.38.5 开发，Dart 语言编写，共 40 个源文件约 6500 行代码，67 个自动化测试全部通过，已发布 Release APK 至 GitHub。整个项目从空白文件夹到可安装的签名 APK 耗时约 28 小时，完整覆盖了技术选型、架构设计、编码实现、测试编写、安全加固和上线发布的全流程。

---

## 功能概述

应用围绕四个核心场景设计：记录、整理、检索、AI 辅助。

**记录**：支持写日记和存知识两种内容类型。日记模块包含 8 种情绪追踪、自动日期分组和标签筛选。知识模块支持 Markdown 编辑、6 大预设分类和共同标签关联推荐。两种内容均支持附件——可直接拍照、从相册选取或导入 PDF、Word、Excel、PPT 等文档，共覆盖 24 种文件格式。附件以缩略图或类型图标展示，点击可由系统应用打开。编辑器内集成了撤销重做系统，基于快照栈实现，最多支持 50 步回退，通过 1.5 秒防抖定时自动保存和操作前手动快照两种策略保护编辑状态。

**整理**：提供长按进入多选模式的批量管理能力，支持全选、取消全选和批量删除，删除前弹出二次确认对话框。数据可通过 JSON 格式全量导出和导入，标签、分类和 AI 摘要数据均包含在内。应用支持浅色、深色和跟随系统三种主题模式，通过 StateNotifier 实现即时切换。

**检索**：基于 SQLite FTS5 扩展自建了全文搜索引擎。6 个数据库触发器自动维护搜索索引——当日记或知识条目发生增、删、改时，触发器自动同步更新索引，应用层无需手动干预。搜索结果以 RichText 红色加粗高亮匹配关键词，附带结果计数。搜索历史最多 20 条持久化存储。

**AI 辅助**：这是应用最核心的差异化能力。项目直接接入了 DeepSeek Chat API，封装了 7 种 AI 调用方法：分析日记生成摘要和标签、分析知识提取关键词、续写内容保持风格一致、润色优化选中段落的表达、将长文总结为要点列表、基于本周所有日记生成回顾报告、以及基于用户个人知识库的 RAG 智能问答。所有 AI 调用通过统一的 Service 层管理，Dio 客户端配置了 15 秒连接超时和 30 秒读取超时，DioException 和通用异常双层兜底确保 API 失败不阻塞主流程。API Key 通过 FlutterSecureStorage 存入 Android Keystore 硬件加密存储，网络传输强制 HTTPS。

RAG 智能问答是项目中技术含量最高的模块。整个流水线自研实现，未使用任何第三方 RAG 框架：用户提问后先通过 FTS5 检索知识库中 Top 5 最相关内容，每条结果截断至 1500 字符防止超出模型上下文窗口，拼接为结构化上下文后连同 system prompt 一起发送至 DeepSeek 进行推理回答，结果以聊天气泡形式展示。system prompt 明确要求模型"如果相关内容不足以回答请诚实说明，不要编造"，避免了 RAG 场景下常见的幻觉问题。聊天气泡使用 SelectableText 渲染，用户可以选中复制 AI 的回答内容。

所有 AI 方法的输入内容均经过长度截断保护——分析类 4000 字、续写 3000 字、润色 2000 字、回顾 8000 字——既是成本控制手段，也防止了超长内容导致 API 调用失败。每类场景有独立的 system prompt，针对具体任务单独调优。

此外，应用集成了语音输入功能。基于 speech_to_text SDK 实现中文语音转文字，包含运行时的麦克风权限检查。权限被拒绝时弹出引导对话框，提供快捷跳转至系统设置页面的入口。相机和相册权限同理。

## 技术实现

项目采用 Feature-First 架构，core 层存放基础设施，features 层按业务模块组织。状态管理使用 Riverpod 2.6.1，路由使用 GoRouter 14.8.0。4 个底部 Tab 通过 StatefulShellRoute 管理，7 个全屏页面统一使用 CustomTransitionPage 实现 FadeTransition 加 SlideTransition 的转场动画效果。

数据库层使用 drift 2.28.2 作为 SQLite ORM，共设计 7 张关系表：日记表、知识表、标签表、分类表、两张多对多关联表和一张键值设置表。标签的增删操作通过单条 SQL 的 INSERT ON CONFLICT DO UPDATE 实现原子 upsert，避免了 read-check-write 模式下的竞态条件。多对多关联表配置了外键 CASCADE 策略，删除日记或知识时自动级联清理标签关联。

数据层之上是 3 个 DAO 类：DiaryDao 提供日记 CRUD、标签筛选 JOIN 查询、情绪分布统计、连续记录天数计算和写作字数统计；KnowledgeDao 提供知识 CRUD、分类筛选、最近条目和基于共同标签的关联推荐；通用 DAO 包含 TagDao 的原子操作和 SettingsDao 的键值存取。

FTS5 全文搜索引擎是数据层的亮点。搜索索引是一张虚拟表，内容由 6 个 SQLite 触发器自动维护。rowid 采用奇偶编码策略——日记条目使用 id×2+1 的奇数 rowid，知识条目使用 id×2 的偶数 rowid——两个表的主键即使相同也不会在搜索索引中冲突。搜索时对用户输入进行分词处理，移除 FTS5 特殊字符防止语法注入，每个词加前缀通配符后执行 MATCH 查询，按 rank 排序返回结果。

## 工程质量

项目建立了完整的测试体系，共 67 个测试用例覆盖 7 个测试文件。单元测试覆盖全部 DAO 方法、FTS5 搜索、标签关联、统计查询和批量删除，边界测试覆盖空标题、超长标题、NULL 字段、主键冲突和 FTS5 同步验证，集成测试覆盖从创建日记到 FTS5 搜索再到删除的全链路流程。flutter analyze 保持零错误零警告。

安全方面，AndroidManifest 关闭了 allowBackup 防止系统备份泄露本地数据库，network_security_config.xml 强制所有网络请求使用 HTTPS，Release 构建启用了 R8 代码混淆和资源压缩，ProGuard 规则保护了 Flutter 引擎、drift 数据库、Dio 网络库以及序列化相关类。

应用已在 GitHub 开源，Release 页面提供三个 CPU 架构的 APK 直接下载：arm64-v8a 版本 20.7MB 覆盖主流 64 位手机，armeabi-v7a 版本 18.5MB 兼容老旧 32 位设备，x86_64 版本 22.0MB 用于模拟器测试。
