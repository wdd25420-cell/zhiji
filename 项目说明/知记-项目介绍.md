# 知记 (Zhiji) — 项目介绍

> Flutter 跨平台 · Agent 首屏 · AI 驱动 · 离线优先 · 个人知识管理 Android 应用  
> GitHub: https://github.com/wdd25420-cell/zhiji

---

## 一、产品定位

知记是一款**AI Agent 驱动的个人知识管理应用**，面向希望用 AI 辅助写作、整理、回顾的用户。核心价值主张是「你的第二个大脑」——帮助用户高效记录日记、管理知识，并通过 AI 自动提取洞察。

**一句话描述**：打开即对话，Agent 帮你写日记、存知识、搜全网，一切都在本地完成，数据不上传云端。

## 二、核心功能

### 2.1 日记模块

| 功能 | 说明 |
|------|------|
| 情绪追踪 | 8 种情绪标签（开心/焦虑/平静/感恩/反思/兴奋/疲惫/期待），emoji 可视呈现 |
| 日期分组 | 自动按「今天/昨天/本周/2025年6月」分组，一目了然 |
| 标签筛选 | 动态加载用户标签，点击 FilterChip 即刻过滤 |
| 批量管理 | 长按进入多选模式，支持全选、批量删除，带二次确认 |
| 草稿保护 | 添加附件时若系统杀进程，下次打开自动恢复未保存内容 |

### 2.2 知识库模块

| 功能 | 说明 |
|------|------|
| 分类浏览 | 6 大预设分类（技术开发/产品设计/AI与机器学习/设计体验/阅读笔记/工具效率），可筛选 |
| Markdown 编辑 | 正文支持 Markdown 语法，详情页完整渲染（标题/列表/代码块） |
| 关联推荐 | 基于共同标签自动推荐相关条目 |
| 附件展示 | 详情页可直接查看图片缩略图、点击文档打开系统应用 |

### 2.3 AI 智能模块（DeepSeek API）

| 功能 | 说明 |
|------|------|
| 自动摘要 | 一键分析日记/知识内容，生成 ≤80 字中文摘要 + 3~5 个关键词标签 |
| 续写 | 基于已有内容 AI 续写下文，风格一致 |
| 润色 | 选中文字 → AI 优化表达，更流畅自然 |
| 总结 | 长文压成要点列表 |
| 每周回顾 | 基于本周所有日记生成情绪模式分析 + 积极建议 |
| 智能问答 (RAG) | 从你的知识库检索相关上下文 → DeepSeek 回答，聊天气泡交互 |

### 2.4 搜索与附件

| 功能 | 说明 |
|------|------|
| 全文搜索 | FTS5 搜索引擎，瞬时分词检索所有日记和知识内容 |
| 搜索高亮 | 匹配关键词红色加粗显示，附「共找到 X 条」计数 |
| 搜索历史 | 最多 20 条搜索记录持久化，点击复用 |
| 附件系统 | 拍照/相册/文档（PDF/Word/Excel/PPT/TXT/ZIP 等），缩略图预览 + 类型图标 |

### 2.5 编辑与体验

| 功能 | 说明 |
|------|------|
| 撤销/重做 | 50 步快照栈，1.5 秒自动保存 + 操作前快照 |
| Markdown 工具栏 | 一键插入加粗/斜体/标题/列表/引用/代码 |
| 语音输入 | 中文语音转文字，运行时权限引导 |
| 深色模式 | 浅色/深色/跟随系统三态，ThemeProvider 即时切换 |
| 转场动画 | 全屏路由统一 FadeTransition + SlideTransition（250ms） |
| 数据导入导出 | JSON 格式全量备份，包含标签、分类、AI 摘要数据 |

### 2.6 隐私与安全

| 措施 | 说明 |
|------|------|
| 全本地存储 | 所有用户数据仅存储在设备 SQLite 中，无后端服务 |
| API Key 加密 | FlutterSecureStorage → Android Keystore 硬件级加密 |
| HTTPS 强制 | network_security_config.xml：`cleartextTrafficPermitted=false` |
| allowBackup | false —— 阻止系统级备份泄露数据库 |
| R8 混淆 | Release 构建开启，isMinifyEnabled + isShrinkResources |
| 隐私政策 | 7 章节完整说明（数据存储/AI服务/附件/导出删除/权限/儿童隐私） |

## 三、技术亮点

### 3.1 架构设计

采用 **Feature-First** 架构，核心模块与业务功能完全解耦：

```
lib/
├── core/          # 基础设施（主题/路由/数据库/网络/共享组件）
└── features/      # 业务模块（首页/日记/知识/AI问答/搜索/设置）
```

**设计原则**：
- 单一数据流：Screen → DAO → drift → SQLite，无多余抽象层
- 导航统一：全项目 `context.push()`，零处 `Navigator.pushNamed`
- 颜色体系：100% 通过 MD3 `ColorScheme`，亮暗双主题自动适配
- 异步安全：所有 async 后 `if (!context.mounted) return`
- AI 安全：所有 prompt 内容长度截断保护，防止超上下文窗口

### 3.2 数据库：FTS5 全文搜索引擎

直接在 SQLite 层实现，无需 Elasticsearch 等外部依赖：

```
FTS5 rowid 编码：日记 id×2+1，知识 id×2（奇偶永不相撞）
6 个自动触发器：INSERT/UPDATE/DELETE × 2 类型
搜索语法：SELECT ... FROM search_index WHERE search_index MATCH ?
```

### 3.3 AI 集成：RAG（检索增强生成）

完整自研 RAG 流水线，不依赖第三方 RAG 框架：

```
用户提问 → FTS5 检索 Top 5 相关内容
        → 每条 body 截断至 1500 字符
        → 拼接为结构化 context
        → 发送至 DeepSeek API（system prompt + context + question）
        → 流式回答展示于聊天气泡
```

### 3.4 草稿保护机制

针对 Android 系统可能在相机/文件选择器等外部 Activity 期间杀进程的特殊场景：

```
启动外部 Activity 前 → _saveDraft() → 全字段序列化到 settings_table
进程被 kill 后 → initState → _restoreDraft() → 自动恢复 + 清除草稿
正常返回 → _clearDraft()
保存成功 → _clearDraft()
```

### 3.5 状态管理

- **Riverpod** 管理全局状态（ThemeMode、Database 异步初始化）
- **StateNotifier** 实现 ThemeMode 即时切换
- **ChangeNotifier** 实现 UndoManager 撤销/重做栈
- GoRouter 的 `StatefulShellRoute` 保持 Tab 状态独立
