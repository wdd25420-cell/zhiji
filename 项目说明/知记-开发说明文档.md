## 一、项目概述

知记 (Zhiji) 是一款 AI Agent 驱动个人知识管理 Android 应用，从零独立开发，使用 Flutter 跨平台框架，集成了 DeepSeek 大语言模型 API。项目规模约 6,500 行 Dart 代码、67 个自动化测试。

### 1.1 开发环境

| 设备 | Windows 11 Pro 24H2 |
|------|---------------------|
| CPU | Intel/AMD x86_64 |
| JDK | JBR JDK 21 (Android Studio 内置) |
| Flutter | 3.38.5 |
| Dart | 3.10.4 |
| Android SDK | 36 |
| 编辑器 | Claude Code + VS Code |

### 1.2 环境配置脚本

```bash
# 必须的环境变量（Bash / Git Bash）
```

### 2.3 架构约定

| 规则 | 原因 |
|------|------|
| 导航用 `context.push()` | GoRouter 在 StatefulShellRoute 内禁止 `Navigator.pushNamed` |
| async 后 `if (!context.mounted) return` | 防止 widget 销毁后 setState 导致内存泄漏 |
| 颜色通过 `ColorScheme` | MD3 亮暗双主题一键切换 |
| DAO 异常向上抛 | 由 Screen 层 try/catch + SnackBar 统一处理 |
| FTS5 触发器自动同步 | 应用层只读不写，避免手动同步遗漏 |
| AI prompt 截断保护 | 超长内容可能导致 API 超 token 限制或高额费用 |

## 三、数据模型设计

### 3.1 ER 图

```
diary_entries ──< diary_tags >── tags
knowledge_entries ──< knowledge_tags >── tags
knowledge_entries >── category_models
settings_table (独立键值存储)
search_index (FTS5 虚拟表，6 触发器自动同步)
```

### 3.2 表结构

#### diary_entries（日记）

| 列 | 类型 | 说明 |
|----|------|------|
| id | INTEGER PK AUTOINCREMENT | 主键 |
| title | TEXT NOT NULL (1-200 字符) | 标题 |
| body_markdown | TEXT DEFAULT '' | 正文（Markdown 格式） |
| emotion | TEXT NULLABLE | 情绪标签名（happy/anxious/calm 等） |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | 最后修改时间 |
| ai_summary | TEXT NULLABLE | AI 生成的摘要 |
| ai_tags | TEXT NULLABLE | AI 生成的标签（逗号分隔） |
| file_paths | TEXT NULLABLE | 附件 JSON 数组 |

#### knowledge_entries（知识）

| 列 | 类型 | 说明 |
|----|------|------|
| id | INTEGER PK AUTOINCREMENT | 主键 |
| title | TEXT NOT NULL | 标题 |
| content_markdown | TEXT | 正文 |
| category_id | INTEGER FK → category_models SET NULL | 所属分类 |
| source_url | TEXT NULLABLE | 原始来源 URL |
| ai_summary | TEXT NULLABLE | AI 摘要 |
| file_paths | TEXT NULLABLE | 附件 JSON |

#### tags（标签）

| 列 | 类型 | 说明 |
|----|------|------|
| id | INTEGER PK AUTOINCREMENT | 主键 |
| name | TEXT UNIQUE NOT NULL | 标签名（唯一） |
| usage_count | INTEGER DEFAULT 0 | 使用次数 |

**原子 upsert**：`INSERT ... ON CONFLICT(name) DO UPDATE SET usage_count = usage_count + 1` 无竞态条件。

#### category_models（分类）

| 列 | 类型 | 说明 |
|----|------|------|
| id | INTEGER PK AUTOINCREMENT | 主键 |
| name | TEXT UNIQUE NOT NULL | 分类名 |
| icon | TEXT | Material Icon 名称 |
| sort_order | INTEGER | 排序权重 |

6 个预设分类于 `onCreate` 批量插入。

#### 多对多关联表

```
diary_tags(diary_entry_id FK CASCADE, tag_id FK CASCADE)  PK = (diary_entry_id, tag_id)
knowledge_tags(knowledge_entry_id FK CASCADE, tag_id FK CASCADE)  PK = (knowledge_entry_id, tag_id)
```

#### settings_table（键值存储）

| 列 | 类型 | 说明 |
|----|------|------|
| key | TEXT PK | 键名 |
| value | TEXT | 值（JSON 或纯文本） |

用途：`diary_draft`、`knowledge_draft`、`search_history`、`theme_mode`

### 3.3 FTS5 全文搜索引擎

```
搜索索引表（虚拟表，内容由触发器自动维护）:
  CREATE VIRTUAL TABLE search_index USING fts5(title, body, source_type, source_id)

6 个自动触发器:
  diary_fts_insert   → INSERT 搜索索引（rowid = id*2+1）
  diary_fts_update   → UPDATE 搜索索引
  diary_fts_delete   → DELETE FROM 搜索索引
  knowledge_fts_*    → 同上（rowid = id*2）

rowid 奇偶分离设计:
  - 日记 rowid = id * 2 + 1（奇数）
  - 知识 rowid = id * 2（偶数）
  - 永不冲突，无需额外的 source_type 判断

搜索语法（防注入处理）:
  1. 移除 FTS5 特殊字符: ^ * " ( )
  2. 分词后为每个词加前缀通配符: "keyword"*
  3. 执行: SELECT ... FROM search_index WHERE search_index MATCH ?
```

## 五、核心功能实现

### 5.1 DeepSeek AI 集成

整个 AI 模块由单一 Service 类封装，对内提供 7 个静态方法：

```
AIService
├── analyzeDiary(title, content)      → (summary, tags)  JSON 结构化返回
├── analyzeKnowledge(title, content)  → (summary, tags)
├── continueWriting(context)          → 续写文本
├── polish(selection)                 → 润色文本
├── summarize(selection)              → 要点总结
├── weeklyReview(contents)            → 周报生成
└── askQuestion(question, context)    → RAG 问答
```

**统一请求封装**：

```dart
static Future<String?> _post(
  String systemPrompt,
  String userContent, {
  int maxTokens = 500,
}) async {
  final response = await dio.post('$baseUrl/v1/chat/completions', data: {
    'model': 'deepseek-chat',
    'messages': [
      {'role': 'system', 'content': systemPrompt},
      {'role': 'user', 'content': userContent},
    ],
    'temperature': 0.5,
    'max_tokens': maxTokens,
    'stream': false,
  }).timeout(const Duration(seconds: 30));
  return response.data['choices'][0]['message']['content'];
}
```

**安全保护**：
- 所有输入通过 `_truncate()` 截断：分析 4000 字、续写 3000 字、润色 2000 字、总结 4000 字、回顾 8000 字
- Dio 超时：连接 15s，读取 30s
- try/catch DioException + 通用异常双层兜底

### 5.2 撤销/重做系统

```
UndoManager (ChangeNotifier)           ← 核心状态，UI 通过 ListenableBuilder 响应
  ├── EditorSnapshot[] _history         ← 快照栈（文本 + 光标位置）
  ├── int _index                        ← 当前位置
  ├── push(text, selection) → 快照入栈，去重，丢弃 redo 分支
  ├── undo() → 返回上一快照
  ├── redo() → 返回下一快照
  └── clear() → 清空

TypingSnapshotTimer                    ← 定时自动快照
  ├── onTextChanged() → 防抖 1.5s → 自动 push
  └── snapshotNow() → 手动立即快照（AI/Markdown/语音/文件操作前）

UndoRedoButtons                        ← UI 按钮对
  └── ListenableBuilder → IconButton(undo/redo)
```

### 5.4 草稿保护

**问题**：Android 系统内存不足时，会在用户使用相机/相册/文件选择器等外部 Activity 期间杀进程，导致 Flutter 的 State 对象被销毁，所有未保存输入丢失。

**解决方案**：

```
┌─────────────────────────┐    ┌──────────────────────────┐
│  编辑器 State            │    │  settings_table          │
│                         │    │                          │
│  打开附件选择器 ──────────►│    │  diary_draft /           │
│                         │    │  knowledge_draft         │
│  _saveDraft()           │    │  (JSON: title/body/      │
│  jsonEncode(全部字段) ────►│    │   emotion/tags/ai/      │
│                         │    │   attachments)           │
│  系统 kill 进程 ☠️        │    │  ← 数据持续存在          │
│                         │    │                          │
│  Widget 重建             │    │                          │
│  initState() ───────────►│    │  getValue('draft')      │
│  _restoreDraft()         │    │  jsonDecode → 恢复全部字段│
│  remove('draft') ───────►│    │  清除草稿                │
└─────────────────────────┘    └──────────────────────────┘
```

### 5.5 RAG 智能问答流程

```
用户输入问题
    │
    ▼
db.search(question) ──────► FTS5 全文检索
    │
    ▼
Top 5 结果 ──────────────► 每条 body 截断 ≤1500 字符
    │
    ▼
结构化拼接 context ──────► "[diary] 标题\n内容\n---\n[knowledge] 标题\n内容"
    │
    ▼
AIService.askQuestion(q, context)
    │
    ▼
DeepSeek API ← system prompt: "你是知记助手，基于以下笔记回答，不编造"
    │
    ▼
聊天气泡渲染 ← SelectableText (可选中复制)
```

### 5.6 批量删除交互

```
长按卡片 → _enterSelectMode(id) → Checkbox 出现 + FAB 隐藏 + AppBar 显示计数
    │
    ├── 点击卡片 → _toggleSelect(id) → 切换选中 / 取消
    │     └── 全部取消 → 自动退出选择模式
    │
    ├── 全选 → _toggleSelectAll → 选中当前筛选下所有条目
    │
    ├── 取消全选 → 清空选中
    │
    └── 点击删除 → AlertDialog 确认 → deleteEntries(ids) → SnackBar 提示
```

### 8.2 关键环境问题与解决方案

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| Gradle loopback 连接失败 | Win11 24H2 内核 Pipe 改用 Unix Domain Socket | 管理员终端 + netsh loopbacklargemtu=disable |
| Selector.open() 崩溃 | 所有 JDK 受 Win11 24H2 影响 | 使用 JBR JDK 21（JetBrains 定制版） |
| _JAVA_OPTIONS 污染 | Claude Code 进程注入 PollSelectorProvider | gradlew.bat 自动清空 _JAVA_OPTIONS |
| Debug APK 174MB | Flutter 默认包含所有 ABI 和调试符号 | Release 分包构建降至 ~20MB |
| keystore 路径解析错误 | key.properties 相对路径基准是 android/app/ | 改为 `../keystore/zhiji.jks` |
