# 知记 App 试用体验评估报告

> 评估日期：2026-06-21
> 评估方式：全量源码逐行审查（45 个 Dart 文件）+ 设计/审计文档交叉验证 + 编译验证

## 一、🔴 用户视角挑出的毛病（按"气人程度"排序）

### A. 流程与导航上的"反直觉"

| # | 问题 | 用户感受 | 代码位置 |
|---|------|---------|---------|
| 1 | **首屏 Agent 对话是冷启动体验**，但首次打开没填 API Key 时，发消息只会得到"发生错误" | 新用户第一印象 = 报错，完全不知道要先去设置填 Key | `chat_screen.dart:181` |
| 2 | **功能入口藏在右下角 FAB 的 BottomSheet 里**，首页/日记/知识库/设置都要"点 FAB → 抽屉 → 选"两步 | 想看日记要点两次，比底部 Tab 多一步，违背"高频功能一步直达" | `app_shell.dart:13` |
| 3 | **从抽屉跳"写日记"用的是 `context.go()` 而非 `push()`** | `go` 会销毁对话历史栈，用户写完日记回不来对话页（`/` 被替换了），导航不一致 | `app_shell.dart:45` |
| 4 | **设置里"深色模式"是循环点击**（系统→浅色→深色） | 不知道下一态是什么，没有任何视觉提示当前切换到哪，专业 App 都用三段开关 | `settings_screen.dart:259` |
| 5 | **首页 AppBar 没有标题居中外的任何功能区入口**，搜索藏在右上小图标 | 与首页"仪表盘"定位不符，发现性差 | `home_screen.dart:25` |

### B. 反馈与状态的缺失

| # | 问题 | 用户感受 |
|---|------|---------|
| 6 | **Agent 思考时只显示静态文字"思考中…"** | 没有"正在搜索知识库→正在联网"的过程感，用户以为卡死了（ChatGPT/Claude 都有打字机式流式输出）|
| 7 | **没有 token/网络错误的具体提示**，只甩 `发生错误: $e` | 用户看到 `DioException [connection timeout]` 一脸懵 |
| 8 | **搜索结果"共找到 X 条"在结果非空才显示**，搜索中无 loading 文案 | 不知道在搜还是在等 |
| 9 | **保存成功的 SnackBar 用 emoji"✅"** + 系统默认样式 | 风格割裂，不是精致 App 的做法 |
| 10 | **知识详情页底部标题写"最近更新"**，实际是"关联推荐"（按共同标签） | 标签误导用户预期 |

### C. 编辑体验的粗糙

| # | 问题 | 用户感受 |
|---|------|---------|
| 11 | **编辑器本质是裸 `TextField` + 一排小图标工具栏** | 写作时没有沉浸感，工具栏图标挤在一起（撤销/语音/附件/Markdown/AI 一行 5 个），手机上容易误触 |
| 12 | **"续写/润色/总结"挤在一个 AI 菜单里**，润色要先选中文本否则才弹 SnackBar 提示 | 操作路径长，且"先选中文本"这个前提无任何引导 |
| 13 | **标题输入框无边框**（`border: InputBorder.none`），正文也无边框 | 整个编辑区像一张白纸，视觉层次缺失，不知道哪里能点 |
| 14 | **字数统计"XX 字"用 `labelSmall`（11px）** 挤在工具栏最右 | 几乎看不清 |
| 15 | **没有自动保存提示**（虽然有 1.5s 草稿，但用户感知不到"已自动保存"） | 用户会反复点"保存"确认 |

### D. 细节毛刺（小但伤质感）

- **情绪选择器**用 emoji 当唯一载体，深色模式下 emoji 背景色块不统一
- **写作热力图**只有 4×7=28 格，且无图例（"深色=写得多"），新手看不懂
- **情绪趋势图**用 fl_chart 但无明显坐标轴/标题，孤零零一张图
- **空状态**文案统一是"点击 + 开始"，但首页 FAB 是抽屉不是"+日记"，文案与入口不匹配
- **Markdown 渲染**没配置代码块语法高亮、引用块样式，长文章阅读体验差
- **`fontFamily` 全程没设置**，DESIGN_SPEC 写了"Söhne/Avenir Next"但代码里没落地，全是系统默认字体 → **这是国产 App 和精品 App 最直观的差距**

## 二、🎯 与市面上精美 App 的差距

### 1. 对标 Flomo（笔记/日记类标杆）—— 差距在"轻"和"回响"

| 维度 | 知记 | Flomo | 差距 |
|------|------|-------|------|
| 启动到记录 | 对话/FAB抽屉/编辑器 多步 | **打开即写**（首屏就是输入框） | 知记重了 |
| 每日回顾 | 无 | **"那年今日"+ 历史推送** | 知记缺情感连接 |
| 写作热力图 | 28 格小方块 | 整年日历 + 徽章 | 知记缺激励闭环 |
| 标签 | 扁平 | **层级标签 `#父/子`** | 知记管理弱 |
| 分享 | 无 | **生成精美卡片图** | 知记无传播 |

### 2. 对标 Notion —— 差距在"编辑器"和"结构化"

| 维度 | 知记 | Notion | 差距 |
|------|------|--------|------|
| 编辑器 | 裸 TextField + Markdown | **块编辑器**（`/` 命令、拖拽排序、callout、toggle） | **代差级** |
| 视图 | 只有列表 | 表格/看板/日历/画廊/时间线 | 单一 |
| 模板 | 无 | 模板库 | 缺新手脚手架 |
| 关联 | 标签扁平 | **Relation/Rollup 双向关联** | 无知识网络 |

### 3. 对标 Day One / 心潮 / 滴水 —— 差距在"氛围"和"多媒体"

| 维度 | 知记 | Day One | 差距 |
|------|------|---------|------|
| 情绪 | 8 emoji 单选 | 情绪 + **趋势分析图表** | 知记有图但糙 |
| 场景 | 无 | **自动定位 + 天气 + 月相** | 缺氛围感 |
| 多媒体 | 仅文档附件 | **照片/视频/音频** | 缺记录维度 |
| 隐私 | 无锁 | **App 锁 + 单篇加密** | 日记无锁是大忌 |

### 4. 综合差距画像

## 三、🎨 UI 界面美化方案（具体到可改的代码）

### 总原则：从"能用"到"想用"的三层升级

### 第一层：色彩与品牌感（投入小，收益大）

**1. 字体是最大的视觉短板。** 立刻落地 DESIGN_SPEC 里规划的字体：

```dart
// app_theme.dart - 现在全默认字体，加：
fontFamily: Platform.isAndroid ? 'SourceHanSansCN' : null,
// 标题单独用更紧的字距：
displaySmall: TextStyle(letterSpacing: -0.5, fontWeight: FontWeight.w700)
```

中文字体建议内置 **思源黑体 / HarmonyOS Sans / MiSans**，正文行高 1.7。

**2. 主色 `#00897B`（青绿）偏冷且暗**，做"知识/日记"这种温暖向的产品，建议：

- 方案 A（保守）：保留青绿，但提高 `primaryContainer` 明度，AI 面板用更暖的浅绿
- 方案 B（推荐）：换 **墨绿+琥珀双色调** —— `primary=#1B4332`、`tertiary=#E76F51`（琥珀做 AI/强调色），更符合"知记"的文人气

**3. AI 相关元素统一视觉语言。** 现在 🤖 emoji、`Icons.auto_awesome`、"AI"文字方块三种混用。建议 **统一一个渐变图标**（紫蓝渐变 `LinearGradient`），所有 AI 入口视觉一致。

### 第二层：布局与组件升级

**4. 首屏重新设计（最关键）。** 建议二选一：

- **方案 A：恢复底部 Tab**（对话/日记/知识/我的），对话作为首屏之一 —— 符合用户习惯，发现性好
- **方案 B：保留 Agent 首屏**，但首屏改成"**今日卡片 + 快捷输入栏**"：顶部一张大卡片（今日心情+日记数+本周字数），下方一个常驻输入框"记录此刻…"，点开才进 AI 对话。这样冷启动（无 API Key）也不会报错

**5. 卡片去圆角混用。** 现在统一规则：**容器 16、中卡 12、小标签药丸 full**。

**6. 对话气泡升级**（现在是纯色块）：

- AI 气泡加细微阴影 + 左侧 4px 主色竖条（像 Claude/ChatGPT 的引用块）
- 用户气泡用 `primary` 实色而非 `primaryContainer`（现在两者区分度低）
- 气泡内文字加"复制"按钮长按弹出

**7. 统计卡片重做**（现在 4 个一样的小卡）。改成 **1 个大主卡 + 3 个小卡** 的不对称布局：

```
┌─────────────────────┐
│ 连续记录 12 天 🔥    │  ← 主卡，渐变背景，大数字
│ 比上周多 3 天        │
└─────────────────────┘
┌────────┐ ┌────────┐ ┌────────┐
│ 日记 42│ │知识 18 │ │字数 2k│
└────────┘ └────────┘ └────────┘
```

### 第三层：动效与微交互（精品感的灵魂）

**8. 加这些动效，质感立刻提升一档：**

- 列表项 **交错入场动画**（`AnimatedList` 或 `Tween` 延迟，每项错开 50ms 上滑+淡入）
- FAB 点击 **转一下 + 缩放**，而不是直接弹 sheet
- 保存成功用 **对勾从圆圈里画出** 的小动画代替"✅ 已保存"文字
- Agent 思考时用 **三个点呼吸** 或 **脑波动画** 代替"思考中…"文字
- 删除日记 **卡片向右滑出** 而不是直接消失

**9. 骨架屏（Shimmer）替代转圈。** 所有 `CircularProgressIndicator`（16 处）在加载列表/卡片时改成骨架屏，这是从"毛坯"到"成品"最直观的一步。

### 第四层：阅读与沉浸

**10. 知识详情页**改成 Notion 式排版：

- 顶部 **大标题 + 元信息条**（分类胶囊、日期、阅读时长估算）
- 正文 `MarkdownStyleSheet` 加大段间距、`h1/h2` 前加主色短线装饰
- 底部"关联推荐"改成 **卡片轮播** 而非 ListTile 列表

**11. 深色模式专门调色。** 现在直接用 `ColorScheme.fromSeed` 自动生成深色，对比度往往不够。建议深色下背景用 `#121212` 纯黑省电，卡片 `#1E1E1E`，文字提一档明度。

## 四、📋 项目优化点与改进优先级路线图

按"性价比（影响÷工作量）"排序，前三项是"改了立刻像换了个 App"的：

### 🚀 第一优先（1 周，性价比最高）

| 优先级 | 改动 | 预期效果 |
|--------|------|---------|
| **P0** | 首屏冷启动友好化：无 API Key 时首屏展示"今日卡片+本地功能"，不报错 | 解决新用户第一印象崩塌 |
| **P0** | 全局加 **中文字体** + 统一 AI 视觉语言 | 视觉质感跃升一档 |
| **P0** | 底部 Tab 导航或首屏大卡片重做 | 解决功能发现性 |
| P1 | 骨架屏替换 16 处转圈 | 加载体验质变 |
| P1 | Agent 流式输出 + 步骤提示（"正在搜索知识库…"） | AI 体验追上 ChatGPT |
| P1 | 设置页深色模式改三段 `SegmentedButton` | 消除反直觉循环点击 |

### 🎨 第二优先（2 周，体验打磨）

- 列表交错入场动画 + 保存/删除微动效
- 对话气泡 + 统计卡片重做（不对称布局）
- 编辑器视觉层次（标题/正文分区 + 工具栏分组收起）
- 写作热力图加图例 + 扩展到全年 + 徽章系统
- 情绪趋势图加坐标轴、标题、交互（点击某天看详情）
- 应用锁（PIN/生物识别）—— 日记类 App 必备

### 💎 第三优先（持续，竞争力）

- Markdown 阅读样式（代码高亮、callout、目录）
- 多媒体：照片/音频附件 + 详情页展示
- 分享卡片生成（截图美化分享）
- 关联推荐升级（标签 + 分类 + 全文相似度）
- 每日回顾推送（那年今日）
- 模板系统（晨间日记、读书笔记、周报模板）

### ⚙️ 工程/技术债（来自既有审计报告，建议穿插修复）

- `getOrCreate` 标签计数原子化（`ON CONFLICT DO UPDATE`）
- RAG 上下文截断（防超 token）— `chat_screen` 搜索结果按字符截断
- 依赖升级（go_router 14→17、riverpod 2→3 等技术债）
- 集成测试 + Widget 测试（当前仅 smoke test）
- `allowBackup` 已改 false（隐私）✅，proguard 补 freezed keep 规则

## 五、总结：一句话评价

> **知记的"骨"很扎实（架构、数据库、Agent 循环、测试都到位），但"皮"还停留在工程师审美阶段。** 离 Flomo/Notion 这类精品 App，差的不是功能数量，而是 **字体、动效、冷启动引导、编辑器沉浸感** 这四件"看不见但摸得着"的细节。优先做 P0 三件（首屏友好化 + 中文字体 + 导航重做），App 的精致度会肉眼可见地提升一个层级。
