一、Agent 本地文件处理能力

用户从系统分享/文件管理器把文件交给 Agent，Agent 自动识别、解析、入库。

文件处理流程
📄 TXT / MD 直接读取，AI 自动分类 + 标签 + 摘要 👤「把这篇笔记导入知识库」
🤖 已导入为📚「XXX」，分类🤖AI
📑 PDF 提取文本，AI 分段总结要点 👤「帮我读这个PDF，总结要点」
🤖 该PDF共12页，核心论点：...
📊 Word / Excel 提取文字/表格，结构化存入 👤「把这个Excel里的数据整理成笔记」
🖼️ 图片 (OCR) 拍照/截图 → OCR识别 → 存为笔记 👤 拍白板 →「把白板内容转成笔记」
🗜️ ZIP 压缩包 解压 → 逐个文件处理 → 批量导入 👤「这个压缩包里的文件全部导入」
🔗 网页链接 抓取网页 → 提取正文 → 存为知识 👤「把这个链接存下来并总结」

三、轻度主动推送（每日一次）

每日定时推送（用户可配置时间，默认 20:00）
📊 今日回顾
今天写了 2 篇日记，1 篇知识。情绪以 😊 开心为主。点击查看详情 →
📚 待复习提醒
有 3 篇笔记超过 30 天未复习。点击查看 →
💡 知识发现
发现「React状态管理」和「Riverpod原理」内容高度相关，要合并整理吗？
🔗 关联推荐
你最近在读的「系统设计」和 3 个月前收藏的文章主题一致。

推送时间可在设置中调整 | 可关闭不想收到的推送类型 | 点击通知直接进入 Agent 对话

四、每日推送技术实现

Android WorkManager 定时任务 → 后台触发 Agent 分析 → 生成本地通知。全程离线，数据不出设备。

20:00 WorkManager 触发
↓
Agent.analyze() 本地执行：
├─ countByDay() → 今日写作统计
├─ getStaleNotes() → 30天+未复习笔记
├─ findRelations() → 新关联发现
└─ countByEmotion() → 情绪数据
↓
flutter_local_notifications 推送
↓
用户点击通知 → 打开 App → Agent 对话页
↓
Agent 基于推送内容展开：「今天过得怎么样？我看到...」
